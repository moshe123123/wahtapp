# גשר וואטסאפ ↔ מייל - גרסת WhatsApp Cloud API הרשמי

בגרסה הזו **אין שום שלב שדורש פתיחה או הורדה של אפליקציית וואטסאפ**.
כל הרישום מתבצע מול ה-API הרשמי של מטא, כולל קבלת קוד אימות (SMS/שיחה) שמזינים
באתר האישי שבנינו.

## שלב 0: הקמת אפליקציה ב-Meta for Developers (חד פעמי, דרך האתר של מטא)
1. גש ל-https://developers.facebook.com/ והתחבר עם חשבון פייסבוק.
2. My Apps → Create App → בחר סוג "Business".
3. בתוך האפליקציה: Add Product → מצא את "WhatsApp" → Set Up.
4. מטא תיתן לך אוטומטית **מספר בדיקה חינמי (Test Number)** + Phone Number ID +
   WABA ID + Access Token זמני - את כל אלה תראה בדף "API Setup" של מוצר הוואטסאפ.
5. **חשוב**: הטוקן שמופיע שם בהתחלה זמני (פג אחרי 24 שעות). כדי לקבל טוקן קבוע:
   Business Settings → Users → System Users → צור System User → הענק לו הרשאות
   על האפליקציה והנכס של WhatsApp → Generate Token (סמן `whatsapp_business_messaging`
   ו-`whatsapp_business_management`).
6. אם אתה רוצה לדבר עם מספר "אמיתי" (לא רק 5 נמענים מאושרים מראש), תצטרך בעתיד
   להוסיף מספר טלפון משלך (סים ייעודי, לא רשום כרגע לאפליקציית WhatsApp הרגילה)
   דרך אותו דף - שם בדיוק מתבצע תהליך הרישום עם קוד האימות.

## שלב 1: הרצה מקומית
```bash
cd server-cloud-api
cp .env.example .env   # מלא עם הפרטים משלב 0
npm install
npm start
```
פתח דפדפן בכתובת `http://localhost:3000` - זהו האזור האישי.

## שלב 2: רישום/אימות מספר דרך האתר האישי
1. מלא את מפתח ה-API (השדה `API_KEY` שהגדרת ב-`.env`).
2. לחץ "שלח לי קוד" (SMS או שיחה) - זה קורא ל-`request_code` של מטא.
3. תקבל קוד בפועל למספר שהוגדר ב-`WHATSAPP_PHONE_NUMBER_ID` → הזן אותו ולחץ "אמת קוד".
4. מזהו המספר "רשום ומאומת" ל-API, ואפשר לשלוח ולקבל הודעות דרכו.

## שלב 3: חיבור ה-Webhook (כדי לקבל הודעות נכנסות)
1. פרוס את השרת ל-Render קודם (ראה למטה) כדי שיהיה לו כתובת ציבורית.
2. ב-Meta for Developers → מוצר WhatsApp → Configuration → Webhook:
   - Callback URL: `https://<your-app>.onrender.com/webhook`
   - Verify Token: אותו ערך שהגדרת ב-`WEBHOOK_VERIFY_TOKEN`
3. הרשם (Subscribe) לשדה `messages`.
4. מרגע זה, כל הודעה נכנסת תישלח אוטומטית למייל שלך (עם plus-addressing לפי מספר השולח).

## שלב 4: פריסה ל-Render
1. העלה את תיקיית `server-cloud-api` לריפו ב-GitHub.
2. Render → New → Web Service → חבר את הריפו.
3. Build: `npm install` | Start: `npm start`.
4. הוסף את כל משתני הסביבה מ-`.env.example` בלשונית Environment.
5. **בשונה מגרסת Baileys - אין כאן צורך ב-Persistent Disk!** כל מצב ה"התחברות" מנוהל
   אצל מטא, לא אצלך, כך שאין session מקומי שיכול להימחק בדיפלוי מחדש.

## שלב 5: תוסף הכרום
זהה לגרסה הקודמת (`extension/`) - אין צורך בשינוי, כי הוא מדבר רק מול ה-`/send`
של השרת שלך, בלי קשר לאיך שהשרת מתחבר לוואטסאפ מאחורי הקלעים.

## מערכת לוגים - איפה רואים בעיות
בכל פעולה מרכזית בשרת (בקשת קוד, אימות מספר, שליחת הודעה, קבלת הודעה, שליחת מייל,
בקשות שנדחו בגלל מפתח API שגוי, ואפילו שגיאות לא צפויות) נרשם לוג עם חותמת זמן.

- **מהאזור האישי** (`/` בדפדפן): יש סקציית "📋 לוגים" עם כפתור "רענן לוגים" וגם
  אפשרות לרענון אוטומטי כל 5 שניות. שגיאות מסומנות באדום, אזהרות בכתום.
- **מהתוסף בכרום**: בפינת הפאנל הצף יש כפתור 📋 שפותח את האזור האישי בשרת (עם
  סקציית הלוגים) בטאב חדש - כך שגם משם, בלחיצה אחת, רואים אם הודעה נתקעה בדרך.
- **גישה ישירה ל-API**: `GET /logs` (עם header בשם `x-api-key`) מחזיר JSON עם כל
  הלוגים; `POST /logs/clear` מנקה אותם.

**חשוב לדעת**: הלוגים נשמרים בזיכרון של השרת בלבד (עד 300 האחרונים) - כל דיפלוי
מחדש או הפעלה מחדש של Render מאפסת אותם. זה מספיק כדי לדבג בעיה "בזמן אמת", אבל
אם תרצה היסטוריה קבועה לטווח ארוך, אפשר בהמשך לחבר שירות חיצוני כמו Logtail/Papertrail.

## מגבלות שכדאי לדעת
- **מצב Test Number**: אפשר לשלוח רק ל-5 מספרי נמענים שאישרת מראש בדשבורד של מטא
  (Recipients list). לשימוש אישי מול קבוצת אנשי קשר קבועה זה בדרך כלל מספיק.
- **חלון 24 שעות**: לפי מדיניות מטא, אפשר לשלוח הודעת טקסט חופשית רק בתוך 24 שעות
  מאז שהצד השני שלח הודעה אליך. מחוץ לחלון הזה נדרשת "תבנית הודעה" (Template)
  מאושרת מראש - רלוונטי בעיקר לשימוש עסקי, פחות לשיחה אישית דו-כיוונית רגילה.
- **שדרוג למספר ייצור אמיתי**: כשתרצה לדבר עם כל מספר (לא רק 5 מאושרים), תזדקק
  למספר טלפון ייעודי (לא רשום כרגע לאפליקציה) ואימות עסקי מול מטא (Business
  Verification) - תהליך שלוקח כמה ימים.
