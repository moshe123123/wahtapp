# גשר וואטסאפ ↔ מייל

פרויקט אישי שמאפשר לשלוח ולקבל הודעות וואטסאפ דרך ממשק מדומה בתוך Gmail,
כאשר כל התעבורה עוברת דרך המייל (ולכן דרך הסינון של נטפרי).

## מבנה הפרויקט
```
server/     שרת Node.js שמחזיק את חיבור הוואטסאפ בפועל (Baileys) + API
extension/  תוסף כרום ל-Gmail שמציג ממשק צ'אט ושולח בקשות לשרת
```

## שלב 1: הרצת השרת מקומית (לבדיקה)
```bash
cd server
cp .env.example .env   # ומלא את הפרטים
npm install
npm start
```
בהרצה הראשונה השרת ידפיס בלוג שנוצר QR. גש לכתובת `http://localhost:3000/qr`,
סרוק עם וואטסאפ בטלפון (הגדרות > מכשירים מקושרים), וזהו - השרת מחובר.

## שלב 2: פריסה ל-Render
1. העלה את תיקיית `server` לריפו ב-GitHub.
2. ב-Render: New → Web Service → חבר את הריפו.
3. Build command: `npm install` | Start command: `npm start`.
4. הוסף את משתני הסביבה מ-`.env.example` בלשונית Environment.
5. **חשוב**: הוסף Persistent Disk (Render → Disks) ומפה אותו לנתיב `/opt/render/project/src/auth_session`
   (או עדכן את `AUTH_FOLDER` ב-`index.js` בהתאם) - אחרת הסשן יימחק בכל דיפלוי ותצטרך
   לסרוק QR מחדש בכל פעם.
6. אחרי הדיפלוי הראשון, גש ל-`https://<your-app>.onrender.com/qr` וסרוק פעם אחת.

## שלב 3: התקנת תוסף הכרום
1. פתח את כרום → `chrome://extensions` → הפעל "מצב מפתח".
2. "טען לא ארוז" (Load unpacked) ובחר את תיקיית `extension`.
3. לחץ על אייקון התוסף ומלא: כתובת השרת מ-Render + ה-`API_KEY` שהגדרת.

## איך זה עובד בפועל
- **הודעה נכנסת** בוואטסאפ → השרת שולח מייל לכתובתך עם `+wa_<מספר>` בכתובת ה-to
  (Gmail plus-addressing), כך שכל שיחה מקובצת לשרשור נפרד. המייל הזה עובר דרך נטפרי
  כרגיל, כמו כל מייל אחר.
- כשאתה פותח את השרשור הזה ב-Gmail, התוסף מזהה סימון נסתר בגוף המייל, ומציג כפתור
  צ'אט צף בפינת המסך.
- הודעה שתשלח מהממשק הצף נשלחת ל-API של השרת (`/send`), שמעביר אותה בפועל לוואטסאפ.

## הרחבות אפשריות להמשך
- חיפוש אנשי קשר: יש כבר endpoint `/contacts` בשרת - אפשר להוסיף לו תיבת חיפוש בתוסף.
- תמיכה בתמונות/קבצים (Baileys תומך בשליחה/קבלה של מדיה - נדרש טיפול נוסף בהעלאה/הורדה).
- החלפת ה-polling הידני ב-Gmail API רשמי (OAuth) במקום הסתמכות על תוכן הדף, לעמידות רבה יותר.
- הוספת livelock: אם ברצונך לשמר את הסינון התוכני שקיים בשירותים המסחריים (למשל רשימת
  מספרים מורשים בלבד), אפשר להוסיף whitelist בצד השרת ב-`/send` וב-`messages.upsert`.
